# The Mirror - Quick Start Guide

## üöÄ Getting Started in 5 Minutes

### Step 1: Start the App

The Mirror is now **fully integrated** with local-first backend. Just run:

```bash
npm install
npm run dev
```

Open `http://localhost:5173`

---

### Step 2: First Reflection

1. Press **Cmd+K** (Mac) or **Ctrl+K** (Windows/Linux) to open the Command Palette
2. Select **"Mirror"** (or type "ref" and press Enter)
3. Start typing your reflection
4. **Wait 2.5 seconds** (constitutional pause detection)
5. Controls appear - click **"Archive"** to save

Your reflection is now:
- ‚úÖ Saved to IndexedDB (persistent)
- ‚úÖ Available in Archive
- ‚úÖ Can be added to Threads
- ‚úÖ Fully exportable

---

### Step 3: Create a Thread

1. **Cmd+K** ‚Üí select **"Threads"**
2. Click **+ button** in threads list
3. Name your thread (e.g., "Morning Thoughts")
4. Press **Enter**

Now all reflections can be linked to this thread!

---

### Step 4: Try Mirrorback (AI Reflection)

1. Open Mirror (Cmd+K ‚Üí Mirror)
2. Type a reflection
3. Wait for pause (2.5s)
4. Click **"Reflect"** button
5. AI generates a constitutional response:
   - ‚úÖ No directives ("you should...")
   - ‚úÖ Questions, not answers
   - ‚úÖ Observations, not prescriptions

---

### Step 5: Export Your Data

You **own all your data**. Export anytime:

1. **Cmd+K** ‚Üí select **"Export"** or **"Data Vault"**
2. Choose format (JSON, Markdown, CSV)
3. Click **"Export"**
4. File downloads to your computer

**Import later:**
1. **Cmd+K** ‚Üí **"Data Vault"**
2. Select "Import"
3. Choose your exported JSON file
4. All data restored!

## üß† Using Identity Axes

### What Are They?

Perspectives from which you reflect. Not roles, but **lenses**.

### Create One:

1. `Cmd+K` ‚Üí "Identity" (or "Self")
2. Click "Add Axis"
3. Name it (e.g., "Parent", "Creative", "Professional")
4. Choose a color
5. Save

### Use It:

When creating a reflection, select an identity axis. View all reflections through that lens later.

## üö® Troubleshooting

### Data Not Saving?

1. Check browser console for errors
2. Ensure IndexedDB is enabled (Settings ‚Üí Privacy)
3. Check available storage space

### Mirrorback Not Working?

Currently using mock AI. Real MirrorOS integration coming soon.

### Export Failed?

1. Check browser allows downloads
2. Try different format (JSON usually works best)
3. Export to smaller chunks if data is large

### Performance Issues?

1. Check number of reflections (>10,000 may slow down)
2. Use search/filters instead of loading all at once
3. Enable "Performance Mode" in Settings
