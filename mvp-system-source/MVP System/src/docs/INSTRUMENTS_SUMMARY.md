**Every instrument enforces:**
- No coercive or directive language

## Core Principles (Never Compromise)

1. **Silence First**
   - Empty states: "Nothing appears here yet."
   - Never: "Get started by creating your first..."

2. **Descriptive, Not Prescriptive**
   - "This will change X to Y"
   - Never: "You should do X"

3. **Equal-Weight Choices**
   - All options same visual prominence
   - Never: Primary/recommended vs secondary

4. **Transparency**
   - Show what data exists, where it goes
   - Constitutional basis for all boundaries
   - Verifiable receipts and checksums

5. **Sovereignty**
   - User controls learning permissions
   - Export everything, anytime
   - Delete without retention
   - Fork and modify the constitution itself

6. **Layer-Aware**
   - Mirror's capabilities change by layer
   - Speech contract enforced
   - No cross-layer data leakage

7. **Failure as Information**
   - Phenomenological failure levels
   - Never blame the user
   - Always show alternatives
   - Constitutional basis for refusals

## Constitutional Test Patterns

Before shipping any instrument, verify:

### 1. Authority Leakage
❓ Does the UI imply "correctness" or "recommended" paths?  
✅ All language is descriptive, not prescriptive

### 2. Pressure Mechanics
❓ Does it create completion urgency or streaks?  
✅ No progress indicators, no time pressure, no rewards

### 3. Default Epistemology
❓ Does it silently decide relevance or importance?  
✅ User controls worldviews, learning, and data usage

### 4. Sovereignty Falsifiability
❓ Can the user verify the system's claims?  
✅ Checksums, receipts, public registries, export capability

### 5. Silence-First
❓ Could this be quieter?  
✅ Empty states are silent, confirmations are brief
